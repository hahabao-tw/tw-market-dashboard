# 資料抓取經驗筆記

記錄這個專案實際驗證過的資料來源、端點、參數與踩過的坑。
未來要新增資料或除錯時,先看這份再動手,可省去重新摸索。

> 所有端點都在 `scripts/fetch_data.py` 裡使用。本筆記與該檔同步維護。

---

## 一、整體架構與通則

- **抓取一律在 GitHub Actions 裡執行**,不是瀏覽器。所以沒有 CORS 問題,可以直接打官方端點。
- **只用 Python 標準庫**(urllib、csv、json),不裝任何套件,確保長期免維護。
- **冪等更新**:每次抓到資料先比對日期,已存在就跳過;假日抓不到新資料就什麼都不寫。所以重複執行、假日執行都安全。
- **時間都用台北時區**(UTC+8)計算;GitHub Actions 的 cron 設定要用 UTC,換算 = 台北時間 − 8 小時。
- **歷史保留**:`KEEP_DAYS = 60` 筆,前端顯示取最近 30 筆。

## 二、期交所(TAIFEX)

### 共通陷阱:Referer 標頭(最重要的一課)

期交所**部分**下載端點會檢查請求的來源頁(Referer),沒帶就回傳一個 HTML 錯誤頁而不是 CSV。

- 症狀:抓到的「CSV」第一格是 `<!DOCTYPE HTML ...`,解析全部失敗。
- 解法:POST 時帶上對應查詢頁的 Referer + `Origin: https://www.taifex.com.tw`。
- 經驗:選擇權端點(optDataDown)**不**檢查 Referer,所以一開始沒帶也成功;
  但三大法人端點(futContractsDateDown)**會**檢查,沒帶就失敗。
  → 結論:所有期交所 POST 一律補上 Referer,最保險。

### 共通:CSV 編碼

期交所 CSV 編碼不固定,可能是 UTF-8 或 Big5。
解法:依序嘗試 `utf-8-sig` → `big5` → `cp950` → `utf-8`。

### 共通:交易時段

報表常同時含「一般交易時段」與「盤後交易時段」兩種資料。
- **盤後時段沒有未平倉量**(該欄是 `-`)。
- 計算未平倉、籌碼相關,**只取「一般」時段**,避免重複計算。

### 共通:日期區間上限

下載查詢有區間長度限制,回補歷史時把區間切成每段 ≤ 25 天分批抓,
每批之間 `sleep(2)` 禮貌性間隔。

### 2-1. 三大法人未平倉(含散戶多空比來源)

- **端點**:`POST https://www.taifex.com.tw/cht/3/futContractsDateDown`
- **Referer**:`https://www.taifex.com.tw/cht/3/futContractsDate`
- **參數**:`queryStartDate`、`queryEndDate`(格式 `YYYY/MM/DD`)、`commodityId=""`(空=全部商品)
- **關鍵欄位**:日期、商品名稱、身份別、多方未平倉口數、空方未平倉口數
- **商品名稱對應**:`臺股期貨`=大台、`小型臺指期貨`=小台、`微型臺指期貨`=微台
- **身份別**:外資、投信、自營商

### 2-2. 全市場未平倉(每日行情,算散戶多空比要用)

- **端點**:`POST https://www.taifex.com.tw/cht/3/futDataDown`
- **Referer**:`https://www.taifex.com.tw/cht/3/futDailyMarketReport`
- **參數**:`down_type=1`、`commodity_id`(TX/MTX/TMF)、`queryStartDate`、`queryEndDate`
- **關鍵欄位**:交易日期、到期月份、未沖銷契約數、交易時段
- **陷阱**:到期月份含 `/` 的是價差組合,要排除;只取「一般」時段。

### 散戶多空比計算公式(估算)

```
散戶多單 = 全市場未平倉 − 三大法人多方OI合計
散戶空單 = 全市場未平倉 − 三大法人空方OI合計
多空比 % = (散戶多單 − 散戶空單) ÷ 全市場未平倉 × 100
```
大台、小台、微台各自計算。此為估算值(全市場減去法人),非官方直接數字。

### 2-3. 選擇權每日行情(最大未平倉)

- **端點**:`POST https://www.taifex.com.tw/cht/3/optDataDown`
- **Referer**:`https://www.taifex.com.tw/cht/3/optDailyMarketReport`
- **參數**:`down_type=1`、`commodity_id=TXO`、`queryStartDate`、`queryEndDate`
- **關鍵欄位**:交易日期、到期月份(週別)、履約價、買賣權、未沖銷契約數、交易時段、**契約到期日**
- **契約到期日欄位**:2025/12/8 起新增,是判斷合約的關鍵,不必再自己解析代碼規則。

#### 合約代碼規則

- 月選:純 6 碼數字,如 `202606`
- 週選:6 碼數字 + `W` + 週數(週三選),如 `202606W2`
- 週五選:6 碼數字 + `F` + 序號(週五選),如 `202606F3`

#### 合約篩選邏輯(只顯示兩個合約)

用「契約到期日」欄位判斷,以「資料日」為基準:
- **近月月選** = 月選中,到期日 ≥ 資料日 的最近一個
- **下一個結算週選** = 週選中,到期日 **嚴格大於** 資料日 的最近一個
  (到期日 = 資料日 的當天結算合約要排除)

範例(資料日 2026/06/12):合約有 `202606F2`(當天到期,排除)、`202606`、
`202606F3`、`202606W4`、`202607`...→ 最終選出 `202606` + `202606F3`。

## 三、證交所(TWSE)

### 共通陷阱:當日 vs 前一日資料

- `openapi.twse.com.tw`(OpenAPI):只有**前一交易日**資料,不適合即時。
- `www.twse.com.tw/rwd/...`:有**當日**資料,參數幾乎相同,優先用這個。

### 共通:民國年轉換

部分報表日期是民國年格式 `115/06/12`,要 +1911 轉西元 → `2026-06-12`。

### 共通:請求頻率

證交所對頻繁請求敏感,逐日回補時每筆之間 `sleep(3)`,務必放慢。

### 3-1. 每日成交統計(成交金額)

- **端點**:`GET https://www.twse.com.tw/rwd/zh/afterTrading/FMTQIK?date=YYYYMM01&response=json`
- **回傳**:整月每日的成交資訊;日期民國年格式。
- **用途**:成交金額(第 3 欄,單位元)。可一次拿整月,適合回補歷史。

### 3-2. 信用交易統計(融資餘額)

- **端點**:`GET https://www.twse.com.tw/rwd/zh/marginTrading/MI_MARGN?date=YYYYMMDD&selectType=MS&response=json`
- **回傳**:單日信用交易;找「融資金額」那一列。
- **欄位**:今日餘額(找表頭含「今日餘額」的欄)、融資買進。單位:仟元。
- **公布時間**:比期交所晚,傍晚以後才齊全 → 排程放 20:30、22:30。

### 3-3. 上市公司總市值(踩坑最久的一個)

- **端點**:`GET https://www.twse.com.tw/rwd/homeApi/mkt_cap`
- **回傳格式**(極簡,無外層包裝):
  ```json
  [["06/10",1410325.63],["06/11",1407898.32],...,["06/16",1494422.39]]
  ```
  每筆 = `["月/日", 總市值]`,**單位已是億元**,近 5 個交易日,最後一筆最新。
- **走過的彎路**:一開始猜總市值在 `MI_INDEX`(大盤統計)裡,結果那張表只有
  成交統計、漲跌證券數,**沒有市值**。後來用瀏覽器 DevTools → Network 分頁,
  在證交所「上市公司總市值」頁(accessibility.twse.com.tw/zh/listed/mkt-cap.html)
  抓到真正的 API 是上面這個 `homeApi/mkt_cap`。
- **限制**:這個 API **只給近 5 日**,無法查指定歷史日期。
  所以「融資槓桿比」只有最近幾天算得出來,更早的歷史是空的,需靠每日累積。

### 融資槓桿比計算

```
融資槓桿比 % = 融資餘額(億) ÷ 上市總市值(億) × 100
```
數量級約 0.3%~0.6%。(注意:不是市值÷融資餘額,那會是約 200 倍而非百分比。)

### 3-4. 發行量加權股價指數(收盤、漲跌幅)

- **端點**:`GET https://www.twse.com.tw/indicesReport/MI_5MINS_HIST?response=json&date=YYYYMM01`
- **回傳**:整月每日 [日期, 開, 高, 低, 收],日期民國年格式。收盤取第 5 欄。
- **漲跌幅**:此端點不直接給,用「(今日收盤 − 前一交易日收盤) ÷ 前一交易日收盤」自算。
  抓當月 + 上月兩個月,確保月初也有前一日可比。

### 3-5. 台積電(2330)收盤價

- **端點**:`GET https://www.twse.com.tw/exchangeReport/STOCK_DAY?response=json&stockNo=2330&date=YYYYMM01`
- **回傳**:整月每日;欄位序為 日期,成交股數,成交金額,開,高,低,**收(第7欄)**,漲跌價差,成交筆數。

### 3-6. 台積電大盤權重(期交所)

- **端點**:`GET https://www.taifex.com.tw/cht/2/weightedPropertion`(HTML 表格,非 JSON)
- **解析**:用 regex 找 `2330` 後最近的百分比即台積電權重(它排第 1)。
- **更新頻率**:每月月底更新一次(權重變動慢),資料來源臺灣指數公司。

### 台積電大盤影響點數(係數)

```
影響點數係數 = 加權指數收盤 × 台積電權重% ÷ 台積電收盤價
```
意義:台積電每跳動 1 元,大盤約被帶動幾點。
範例:45877.39 × 41.992% ÷ 2385 ≈ 8.08 點。
此項只在每天 20:00 更新一次(證交所盤後資料齊全後)。

## 四、排程(GitHub Actions)

cron 用 UTC,平日(週一~五)執行:

| 台北時間 | UTC cron | 抓什麼 |
|---|---|---|
| 14:55 | `55 6 * * 1-5` | 期貨三大法人、選擇權 |
| 15:15 | `15 7 * * 1-5` | 期貨、選擇權(補抓) |
| 15:40 | `40 7 * * 1-5` | 選擇權(補抓) |
| 17:00 | `0 9 * * 1-5` | 期貨保險班次 |
| 20:30 | `30 12 * * 1-5` | 證交所融資、市值 |
| 22:30 | `30 14 * * 1-5` | 證交所融資、市值(補抓) |

- 免費版排程實際執行可能延遲 5~15 分鐘,因冪等設計不影響正確性。
- `workflow_dispatch` 可手動觸發;手動觸發時帶 `FORCE=1`,可在假日強制執行。
- repo 連續 60 天無活動,GitHub 會暫停排程並寄信,進去點按鈕即恢復。

## 五、除錯心法

1. **先看 log**:腳本在關鍵處都有 `log()`,第一次跑或出錯時 log 會印出
   CSV 表頭、API 原始回傳前幾百字、選定的合約等,據此判斷問題。
2. **抓到 HTML 而非 CSV** → 幾乎都是缺 Referer(見二之共通陷阱)。
3. **欄位找不到** → 官方可能改了欄位名,看 log 印出的表頭,調整 `col_index` 關鍵字。
4. **數字怪異** → 檢查單位(元/仟元/億元)與民國年轉換。
5. **要找新 API** → 用 Chrome DevTools(F12)→ Network 分頁 → F5 重整 →
   找 xhr/fetch 類型的請求 → 看 Headers 的 Request URL、Response 的回傳內容。
   這是找出官方頁面背後資料來源最可靠的方法。
